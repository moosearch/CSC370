a) [ ] Spanned records can be used to store variable length records because a pointer is placed at the index to point to the blocks where the record is continued.

b) [ ] Weak entity types may not have any key attributes of their own, and hence may not be able to distinguish between some entities because combinations of values of these attributes are identical.

c) [ ] The cardinality ratio constraint in an entity-relationship diagram, specifies the number of relationship instances in which an entity can participate.

d) [ ] The Cartesian product is a binary set operation on relations which do not need to be union compatible, and is used to combine tuples from two relations so that related tuples can be identified.

e) [ ] A distributed database system can have the actual database, but not the DBMS software, distributed over many widely-dispersed sites connected by a computer network.

f) [ ] The dummy record type is a record type that is added to the network model to facilitate the representation of M:N relationships.

g) [ ] A host language is a high-level programming language such as, COBOL, Pascal, PL/I, etc., that is used to hold embedded SQL or other query language commands.

h) [ ] A lossless join ensures that no spurious tuples are generated when a natural join is applied to relations in a decomposition. The word lossless applies to the loss of information, not the loss of tuples.

  1. i) [ ] A nondense (or sparse) index for a file is an index that contains an entry for every data block in the file.

j) [ ] Indexed sequential access is a term used to describe situations where the user wants indexed access to records, and sequential access, ordered by key, on those same records. This is usually implemented by using B+ trees.

k) [ ] A file that has a secondary index on every one of its fields is called a fully inverted file. Because all indexes are secondary, new records are inserted using a composite primary index.

l) [ ] A functional dependency X Y is a partial functional dependency if there is some attribute A (that is a member of X) that can be removed from X and the dependency will no longer hold.

m) [ ] A repeating group in the network model allows the inclusion of a set of composite values for a data item in a single record.

n) [ ] A virtual data item is stored automatically in a record because it is derived from actual data items using some procedure that is defined specially for that purpose.

o) [ ] An Occurrence tree is used in the network model to mark the current record types and sets so that they can be updated or deleted quickly by a subsequent command.

Entity-Relationship Schema for a Pharmacy Data Model

The following is a data description of an entity-relationship schema for a Pharmacy data model.

Entities and Attributes:

Drug : DName, DCode. The drug name and drug code are a medication prescribed by a physician to a patient.

Drug-Brand: DbName, DbUsage,DbPrice. A Drug Supplier supplies a given drug under a Drug Brand Name at the unit price in DbPrice, and keeps track of the total year-to-date quantity of that drug used in DbUsage.

Drug-Supplier: DsID, DsName. These are the name and identification number of the company supplying the drug.

Patient: PID, PFName, PLName, PType. The first name, last name, identification number and patient type code of a person receiving treatment.

Pharmacist: PhID, PhLName, PhFName. The first name, last name and identification number of the pharmacist who is filling prescriptions.

Physician: PyLName, PyFName. The first name and last name of the physician who is issuing (prescribing) the prescriptions.

Prescription: PrID, PrDate, PrDrug, PrQty, PrInstr_Text. The identification (prescription) number, date, drug being prescribed, the quantity being issued, and the instructions for use.

Relationships:

A patient can use no or many prescriptions, but a prescription can be issued only to one patient. A prescription cannot exist without a patient.

A pharmacist can fill no or many prescriptions, but a prescription can be filled by only one pharmacist.

A prescription can exist without a pharmacist because a physician writes a prescription before it is filled.

A physician can issue many prescriptions, but a prescription can be issued by only one physician. A prescription cannot exist without a physician.

A drug brand can be prescribed by many prescriptions, but a prescription can prescribe only one drug brand. A prescription cannot exist without a drug brand.

A drug can be supplied by many drug suppliers, and a drug supplier can supply many drugs. The Drug_Brand entity is the combination of drug and drug supplier.
